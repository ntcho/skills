# POST /audiences/{audience_id}/contacts

**Resource:** [audiences](../resources/audiences.md)
**Add Contact**
**Operation ID:** `createAudienceContact`

Create a new omni-channel contact for an audience.

## Parameters

| Name | In | Type | Required | Description |
|------|------|------|----------|-------------|
| `audience_id` | path | string | Yes | The unique ID for the audience. |
| `merge_field_validation_mode` | query | enum: ignore_required_checks, strict | No | Defines how merge field validation is handled. When set to `ignore_required_checks`, the API does not raise an error if required merge fields are missing from the request. When set to `strict`, the API enforces validation and returns an error if any required merge field is not provided. If this setting is omitted, `strict` is applied by default. |
| `data_mode` | query | enum: historical, live | No | Indicates the data processing mode. In `historical` mode, contact data changes do not trigger automations or webhooks. In `live mode`, such changes do trigger them. |

## Request Body

**Required:** Yes

**Content Types:** `application/json`

## Responses

| Status | Description |
|--------|-------------|
| 200 |  |
| default | An error generated by the Mailchimp API. |
