# GET /lists/{list_id}/members

**Resource:** [lists](../resources/lists.md)
**List members info**
**Operation ID:** `getListsIdMembers`

Get information about members in a specific Mailchimp list.

## Parameters

| Name | In | Type | Required | Description |
|------|------|------|----------|-------------|
| `fields` | query | string[] | No | A comma-separated list of fields to return. Reference parameters of sub-objects with dot notation. |
| `exclude_fields` | query | string[] | No | A comma-separated list of fields to exclude. Reference parameters of sub-objects with dot notation. |
| `count` | query | integer | No | The number of records to return. Default value is 10. Maximum value is 1000 |
| `offset` | query | integer | No | Used for [pagination](https://mailchimp.com/developer/marketing/docs/methods-parameters/#pagination), this is the number of records from a collection to skip. Default value is 0. |
| `list_id` | path | string | Yes | The unique ID for the list. |
| `email_type` | query | string | No | The email type. |
| `status` | query | enum: subscribed, unsubscribed, cleaned... | No | The subscriber's status. |
| `since_timestamp_opt` | query | string | No | Restrict results to subscribers who opted-in after the set timeframe. Uses ISO 8601 time format: 2015-10-21T15:41:36+00:00. |
| `before_timestamp_opt` | query | string | No | Restrict results to subscribers who opted-in before the set timeframe. Uses ISO 8601 time format: 2015-10-21T15:41:36+00:00. |
| `since_last_changed` | query | string | No | Restrict results to subscribers whose information changed after the set timeframe. Uses ISO 8601 time format: 2015-10-21T15:41:36+00:00. |
| `before_last_changed` | query | string | No | Restrict results to subscribers whose information changed before the set timeframe. Uses ISO 8601 time format: 2015-10-21T15:41:36+00:00. |
| `unique_email_id` | query | string | No | A unique identifier for the email address across all Mailchimp lists. |
| `vip_only` | query | boolean | No | A filter to return only the list's VIP members. Passing `true` will restrict results to VIP list members, passing `false` will return all list members. |
| `interest_category_id` | query | string | No | The unique id for the interest category. |
| `interest_ids` | query | string | No | Used to filter list members by interests. Must be accompanied by interest_category_id and interest_match. The value must be a comma separated list of interest ids present for any supplied interest categories. |
| `interest_match` | query | enum: any, all, none | No | Used to filter list members by interests. Must be accompanied by interest_category_id and interest_ids. "any" will match a member with any of the interest supplied, "all" will only match members with every interest supplied, and "none" will match members without any of the interest supplied. |
| `sort_field` | query | enum: timestamp_opt, timestamp_signup, last_changed | No | Returns files sorted by the specified field. |
| `sort_dir` | query | enum: ASC, DESC | No | Determines the order direction for sorted results. |
| `since_last_campaign` | query | boolean | No | Filter subscribers by those subscribed/unsubscribed/pending/cleaned since last email campaign send. Member status is required to use this filter. |
| `unsubscribed_since` | query | string | No | Filter subscribers by those unsubscribed since a specific date. Using any status other than unsubscribed with this filter will result in an error. |

## Responses

| Status | Description |
|--------|-------------|
| 200 |  |
