# POST /customer-journeys/journeys/{journey_id}/steps/{step_id}/actions/trigger

**Resource:** [customerJourneys](../resources/customerJourneys.md)
**Customer Journeys API trigger for a contact**
**Operation ID:** `postCustomerJourneysJourneysIdStepsIdActionsTrigger`

A step trigger in an Automation flow. To use it, create a starting point or step from the Automation flow builder in the app using the Customer Journey API condition. We’ll provide a url during the process that includes the {journey_id} and {step_id}. You’ll then be able to use this endpoint to trigger the condition for the posted contact.

## Parameters

| Name | In | Type | Required | Description |
|------|------|------|----------|-------------|
| `journey_id` | path | integer | Yes | The id for the flow. |
| `step_id` | path | integer | Yes | The id for the Step. |

## Request Body

**Required:** Yes

**Content Types:** `application/json`

## Responses

| Status | Description |
|--------|-------------|
| 204 | An empty response |
| default | An error generated by the Mailchimp API. |
