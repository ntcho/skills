# POST /lists/{list_id}

**Resource:** [lists](../resources/lists.md)
**Batch subscribe or unsubscribe**
**Operation ID:** `postListsId`

Batch subscribe or unsubscribe list members.

## Parameters

| Name | In | Type | Required | Description |
|------|------|------|----------|-------------|
| `list_id` | path | string | Yes | The unique ID for the list. |
| `skip_merge_validation` | query | boolean | No | If skip_merge_validation is true, member data will be accepted without merge field values, even if the merge field is usually required. This defaults to false. |
| `skip_duplicate_check` | query | boolean | No | If skip_duplicate_check is true, we will ignore duplicates sent in the request when using the batch sub/unsub on the lists endpoint. The status of the first appearance in the request will be saved. This defaults to false. |

## Request Body

**Required:** Yes

**Content Types:** `application/json`

## Responses

| Status | Description |
|--------|-------------|
| 200 |  |
| default | An error generated by the Mailchimp API. |
